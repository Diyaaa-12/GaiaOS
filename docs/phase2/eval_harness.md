# Evaluation Harness & Benchmark Questions

The evaluation harness in GaiaOS provides a structured framework for measuring the correctness, latency, and calibration of the orchestrator and domain agents.

## Overview

The evaluation suite runs benchmark questions saved in the PostgreSQL `eval_benchmark_questions` table. The runner executes these queries, scores response correctness, and logs the score and metrics to `eval_benchmark_runs`.

## DB Schema

### `eval_benchmark_questions`

Holds the curated golden dataset of test questions:
- `id`: UUID (Primary Key, autogenerated).
- `question_text`: Text query submitted to the platform.
- `expected_domains`: Array of strings indicating the agent domains expected to handle the query (e.g. `['air_quality']`, `['seismic', 'ocean']`).
- `expected_complexity`: Routing complexity tier classification (`trivial`, `moderate`, or `complex`).
- `reference_answer`: The expected correct answer or baseline explanation.
- `reference_evidence`: JSONB block representing the expected supporting evidence items.

### `eval_benchmark_runs`

Holds historical run metrics for regression analysis:
- `id`: UUID (Primary Key).
- `benchmark_question_id`: Foreign Key referencing the question.
- `investigation_id`: Nullable UUID referencing the generated investigation (linked when running real workflows in Milestone 2+).
- `orchestrator_version`: A git-SHA or version tag identifier of the codebase running the suite.
- `score`: Numeric (3,2) evaluation score.
- `metrics`: JSONB containing step-by-step duration, agent outputs, and detailed error summaries.
- `run_at`: Timestamp when the benchmark question ran.

## How to Add a New Benchmark Question

To add a benchmark question, you must insert a record into the `eval_benchmark_questions` table.

### Option A: Via Alembic Migration (Recommended for versioned golden sets)

Create a new Alembic migration:
```bash
alembic revision -m "add_beijing_air_quality_benchmark"
```

In the migration's `upgrade()` function, execute an INSERT statement:
```python
def upgrade() -> None:
    op.execute(
        """
        INSERT INTO eval_benchmark_questions (
            id,
            question_text,
            expected_domains,
            expected_complexity,
            reference_answer,
            reference_evidence
        )
        VALUES (
            'b0e0b0e0-b0e0-b0e0-b0e0-b0e0b0e0b0e0',
            'What are the PM2.5 levels in Beijing?',
            ARRAY['air_quality'],
            'trivial',
            'Beijing PM2.5 is moderate at 45 ug/m3.',
            '{"source": "OpenAQ", "pm25": 45}'::jsonb
        );
        """
    )

def downgrade() -> None:
    op.execute("DELETE FROM eval_benchmark_questions WHERE id = 'b0e0b0e0-b0e0-b0e0-b0e0-b0e0b0e0b0e0';")
```

Apply migrations:
```bash
alembic upgrade head
```

### Option B: Direct SQL insert
Connect to PostgreSQL and run the SQL query directly:
```sql
INSERT INTO eval_benchmark_questions (
    question_text,
    expected_domains,
    expected_complexity,
    reference_answer,
    reference_evidence
)
VALUES (
    'Are there any earthquakes above magnitude 5 in California?',
    ARRAY['seismic'],
    'trivial',
    'A magnitude 5.2 earthquake occurred near Los Angeles on July 10.',
    '{"source": "USGS", "magnitude": 5.2}'
);
```

## Executing the Suite

To run the benchmark suite programmatically or in a CI step, import and call `run_benchmark_suite` from python:

```python
import asyncio
from eval.harness.runner import run_benchmark_suite

async def main():
    result = await run_benchmark_suite(
        orchestrator_version="1.0.0-alpha.1"
    )
    print(f"Executed {result.total_questions} questions. Success count: {result.successful_runs}")

if __name__ == "__main__":
    asyncio.run(main())
```
